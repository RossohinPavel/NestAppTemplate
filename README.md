# NestAppTemplate

Шаблон NestJs приложения под разные нужды.


## Стэк

- pnpm - Используется этот менеджер пактов.
- typescript - Само собой, последней версии.
- Nest - Также, последней версии.
- Fastify адаптер. Стандартное ядро Nest заменено на `Fastify` - более быструю альтернативу.
- Typia - Используется как валидатор DTO. Экстремально быстрый. Пишем типы на тайпскрипте, 
валидируем объекты в любом месте. Профит.
- Nestia - обертка над Typia, которая позволяет типизировать ручки в контроллерах и использовать
шуструю валидацию от Typia. Тянет за собой еще и автогенератор Swagger-документации.


# Swagger-документация.

Шаблон предполагает, что на основе его будут создаваться публичные апи. Это означает, что любой 
желающий сожет получить спецификацию всех ручек приложения.
Документация генерируется автоматически во время запуска приложения и доступна по адресу `/docs`.


# Использование

Предполагается, что эту репу форкнут и/или будут использовать как шаблон для создания рабочих приложух.
Соответственно, стянуть проект, написать `pnpm start:dev` и на `localhost:3000` увидеть `Hello world!`.


# Примечания

- Приложение жестко запускается на `0.0.0.0:3000`. Сделано это для удобства работы с докером.
Если такой расклад не устраивает, вы всегда можете поменять это на свое усмотрение.
- Логгирование разделено на обработку непосредственно фреймворка и сервисных вызовов. Для логгирования
сообщений от фреймворка исопльзуется встроенный в NestJS логгер. Он сконфигурирован в `src/main.ts`
и будет выводить в консоль все запросы к фреймворку и результат их обработки. Настройка логгирования 
сервисных вызовов должна производится самостоятельно. Для этого подойдут любые средства, как встроенный
логгер, так и `console.`. Настоятельно советую использовать разное форматирование для сообщений 
фреймворка и сервиса.
- Привычного `Prettier` в проекте нет. Форматированием кода занимается плагин для линтера. Соответственно,
команда `pnpm lint` выполнит как проверку так и форматирование кода. Настройки можно посмотреть 
в соответствующем файле. Там указаны комментарии к правилам.
